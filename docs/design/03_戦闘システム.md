# 03 戦闘システム

## 1. v0 の方針

> **オフェンス1 + ディフェンス1 の同時決定 → 自動再生**

最小構成で「噛み合いの面白さ」が成立するかを確認することを最優先にする。
複雑化は v0.x で段階的に。

## 1.5. 試合構造（ラウンドとターン）

**3ラウンド制。1ラウンド = 3:00 = 12ターン。1ターン = 15秒進行。**

```
試合
├─ ラウンド1 (12ターン = 3:00)
│   └─ インターバル (1:00、HP/スタミナ少回復)
├─ ラウンド2 (12ターン)
│   └─ インターバル
└─ ラウンド3 (12ターン)
    └─ 最終判定（KO していなければ HP 残量で勝敗）
```

### 設定値の保持

```yaml
match_config:
  rounds: 3
  turn_seconds: 15
  turns_per_round: 12          # 試合最大ターン数 = 36
  interval_seconds: 60
```

## 1.6. ダウン・カウント・3ノックダウン制

### 基本ルール

- HEAD HP か BODY HP のどちらかが 0 → **ダウン発生**
- 1ラウンドに **3回ダウン** で **TKO負け**
- ダウン中は **10カウント**。間に立ち上がれなければ **KO負け**
- KDカウントは **ラウンド開始時に0リセット**

### 立ち上がり判定式（暫定）

立ち上がれる確率は HEAD と BODY の残量が **両方とも低い** ほど低くなる。

```
get_up_chance = base_chance
              * (head_hp + recovery_buffer) / head_hp_max
              * (body_hp + recovery_buffer) / body_hp_max
              * personality_modifier
              * round_modifier
```

簡易の数値感:

| 状況 | 立ち上がり率 |
|---|---|
| HEAD 0 ／ BODY 70% | 〜85% |
| HEAD 0 ／ BODY 40% | 〜55% |
| HEAD 0 ／ BODY 10% | 〜20% |
| HEAD 0 ／ BODY 0  | 〜5%（実質 KO 寸前） |

### 立ち上がり後の回復

成功した場合、HEAD/BODY HP が **一定割合回復**：

```
head_hp = max(head_hp + recovery_amount, 1)
body_hp = max(body_hp + recovery_amount, 1)
```

## 2. コマンド体系（v0、2026-05-10 確定）

### 2.1. オフェンス: 8種

**威力順: ジャブ < ストレート < フック < アッパー**（HEAD/BODY それぞれ）

| 技 | 部位 | 威力 | 命中（基準） | 速度 | スタミナ | 距離適性 |
|---|---|---|---|---|---|---|
| ジャブ | HEAD | 小 | 高 | 高 | 小 | **全距離安定** |
| ストレート | HEAD | 中 | 中 | 中 | 中 | **遠距離有利** |
| フック | HEAD | 大 | 低 | 低 | 大 | **中距離有利** |
| アッパー | HEAD | 特大 | 中 | 低 | 大 | **近距離有利** |
| ボディジャブ | BODY | 小 | 高 | 高 | 小 | 全距離安定 |
| ボディストレート | BODY | 中 | 中 | 中 | 中 | 遠距離有利 |
| ボディフック | BODY | 大 | 低 | 低 | 大 | 中距離有利 |
| ボディアッパー | BODY | 特大 | 中 | 低 | 大 | 近距離有利 |

### 2.2. ディフェンス: 5種

| 技 | 効果 | 距離変化 | 備考 |
|---|---|---|---|
| **ガード上** | HEAD攻撃に被ダメ **0**。BODY攻撃には効かず通常ダメージ | 変化なし | 部位読み合い |
| **ガード下** | BODY攻撃に被ダメ **0**。HEAD攻撃には効かず通常ダメージ | 変化なし | 部位読み合い |
| **ダッキング** | HEAD/BODY両方を回避（HEADの方が回避率高）／**失敗時は通常被弾＆距離変化なし** | **成功時のみ1段近づく** | 接近して次にアッパー、の流れ |
| **スウェー** | HEAD/BODY両方を回避（HEADの方が回避率高）／**失敗時は通常被弾＆距離変化なし** | **成功時のみ1段離れる** | 離れて次にストレート、の流れ |
| **カウンター** | 相手のオフェンスに反応して**どちらかが当たる**（成功＝相手に1.3倍／失敗＝自分が1.3倍）。BODY系の方が成功しやすい | 変化なし | リスクリターン型 |

### 2.3. コマンド規模

オフェンス8 × ディフェンス5 = **40通り** の噛み合わせ。
これをマトリクスから **計算式で出す** ことで実況パターンも生成する。

## 2.5. 1ターンの解決構造（2026-05-10 確定）

各プレイヤーは **オフェンス1 + ディフェンス1** を選ぶ。
**オフェンスは常に発動する**（自分のオフェンスは相手のディフェンスとぶつかる）。
**ディフェンスは相手の攻撃を受け止めるための独立枠**。

```
ターン開始
  ├─ プレイヤー選択: オフェンスX + ディフェンスY
  └─ 敵選択:        オフェンスZ + ディフェンスW
        ↓ 同時公開
  ┌─ 解決A: プレイヤーのオフェンスX  ←  敵のディフェンスW で受ける
  └─ 解決B: 敵のオフェンスZ           ←  プレイヤーのディフェンスY で受ける
        ↓
  HP / 距離 / リングポジション / KDカウントを反映
```

### 解決順序（速度差 + 確率）

speed が高い方が **先攻**になりやすい。完全固定ではなく確率で揺れる：

```
p_player_first = 0.5 + (player.speed - enemy.speed) * 0.05
                # speed 差 +5 で +25% 先攻補正、最大 0.9 / 最小 0.1 でクランプ
```

先攻が KO（ダウン発生）させた場合でも、後攻のオフェンスは **同ターン内で発動する**
（同時に殴り合った相打ち KO 演出が起きうる）。
> **TBD**: ダウン中に後攻が殴れるかは別途。仮: ダウン後の攻撃は無効（人道的なルールに合わせる）。

### ディフェンス種別ごとの解決ルール

#### ガード上 / ガード下
- 攻撃部位と一致 → **被ダメ 0**
- 攻撃部位と不一致 → **通常ダメージ**（クリーンボーナスなし）
- 距離変化なし

#### ダッキング / スウェー
- **回避判定**: HEAD攻撃の回避率 > BODY攻撃の回避率
- **回避成功時**: 攻撃を完全に避け、距離が 1段 動く（ダッキング=近づく / スウェー=離れる）
- **回避失敗時**: 通常被弾、**距離変化なし**（動こうとしたが食らった）

#### カウンター
- 相手のオフェンスに対し成功率判定
- **成功**: 相手の攻撃が空振り、相手に **1.3倍ダメージ** で打ち返す（カウンター成功時の威力は相手のオフェンスの基準値で計算）
- **失敗**: 通常被弾の **1.3倍ダメージ** を食らう
- BODY攻撃に対する成功率の方が高い
- 距離変化なし

### 両者の動きが重なった場合

距離変化は **両者の動きを独立に累積適用** する：

| プレイヤー | 敵 | 結果 |
|---|---|---|
| ダッキング成功 | ダッキング成功 | プレイヤー1段近づく + 敵1段近づく = **2段近づく** |
| スウェー成功 | スウェー成功 | プレイヤー1段離れる + 敵1段離れる = **2段離れる** |
| ダッキング成功 | スウェー成功 | プレイヤー1段近づく + 敵1段離れる = **±0** |

ただし距離は **クリンチ〜離れ** の範囲でクランプ。詰めきった場合は §2.7 ポジション変化に流れる。

## 2.6. カウンターの仕組み（明確化）

> オフェンスとディフェンスは **独立した別枠**。カウンターを選んでも自分のオフェンスは普通に発動する。

### 解決例

| シーン | 解決A: プレイヤーのオフェンス vs 敵のディフェンス | 解決B: 敵のオフェンスZ vs プレイヤーのディフェンスY |
|---|---|---|
| プレイヤー: ストレート + ガード上 / 敵: ボディフック + カウンター | プレイヤーのストレート vs 敵カウンター → 成功なら敵が1.3倍ダメ食らわす（プレイヤーへ）／失敗ならストレートが敵に1.3倍ヒット | 敵のボディフック vs プレイヤーのガード上 → 部位ハズレで通常ダメージ |
| プレイヤー: ジャブ + カウンター / 敵: フック + カウンター | プレイヤーのジャブ vs 敵カウンター → カウンター判定 | 敵のフック vs プレイヤーのカウンター → カウンター判定 |

両者カウンターでも、**両方のオフェンスが普通に発動** し、それぞれが相手のカウンターと判定される。

## 2.7. リングポジション（2026-05-10 新規）

距離（相対）とは別に、**リング上の絶対位置** を各プレイヤーごとに管理する：

| ポジション | 説明 |
|---|---|
| **中央** | リング中央付近 |
| **ロープ際** | ロープを背負っている、限定的に動きづらい |
| **コーナー際** | コーナーに追い込まれた、最も不利 |

### ポジション遷移トリガー

距離が **クリンチ** まで詰まった状態でさらに押し合いになると、**押された側のポジション** が1段後退する：

```
中央 → ロープ際 → コーナー際
```

具体的には：

- 距離がすでにクリンチ／近距離で、相手がさらに近づこうとした（ダッキング成功 etc.） → 自分のポジションが1段後退
- 距離が離れ過ぎる方向で、自分が下がりきった（リング外には出られない） → 自分のポジションが1段後退

### ポジションの効果

| ポジション | 効果 |
|---|---|
| 中央 | 補正なし |
| ロープ際 | 命中 -10、回避系（ダッキング/スウェー）の成功率 -15% |
| コーナー際 | 命中 -25、回避系の成功率 -30%、ガード以外のディフェンスが弱い |

### ポジション回復

- ジャブ/ストレートのヒットで反対方向に押し返す（条件付き）
- インターバルでリセット（ラウンド開始時は両者中央）

## 3. 噛み合わせマトリクス（叩き台）

最終ヒット率と威力は次の **3レイヤー** の積で求める：

1. **レイヤーA: 部位の読み合い**（ガード上/下 vs 攻撃部位）
2. **レイヤーB: 距離適性**（攻撃技 vs 現在の距離）
3. **レイヤーC: 技 × 防御技の相性**（ダッキング/スウェー/カウンター系）

### レイヤーA: ガードの読み合い

| 攻撃 \ 防御 | ガード上 | ガード下 | ガード以外 |
|---|---|---|---|
| HEAD系 | **被ダメ 0** | 通常ヒット | レイヤーCで判定 |
| BODY系 | 通常ヒット | **被ダメ 0** | レイヤーCで判定 |

### レイヤーB: 距離適性（命中補正）

距離は **5段階**: クリンチ ／ 近 ／ 中 ／ 遠 ／ 離れ
（リングポジション = コーナー際/ロープ際は別レイヤーで処理、§2.7 参照）

| 技 | クリンチ | 近 | 中 | 遠 | 離れ |
|---|---|---|---|---|---|
| ジャブ系 | -10 | ±0 | ±0 | ±0 | ±0 |
| ストレート系 | -30 | -10 | ±0 | **+15** | -10 |
| フック系 | -10 | ±0 | **+15** | -10 | -30 |
| アッパー系 | ±0 | **+15** | ±0 | -20 | -40 |

### レイヤーC: 技 × ダッキング/スウェー/カウンター

| 攻撃 \ 防御 | ダッキング 回避率 | スウェー 回避率 | カウンター 成功率 |
|---|---|---|---|
| HEAD系 | 高（仮 60%） | 高（仮 60%） | 低（仮 30%） |
| BODY系 | 中（仮 30%） | 中（仮 30%） | **高**（仮 50%） |

> 数値は仮。Q14〜Q16 で詰める。

### 計算順

```
final_hit  = base_hit
           + matrixA  (ガード=0/通常)
           + matrixB  (距離適性)
           + position_penalty  (ロープ際/コーナー際)
           + (atk.speed - def.speed) * 0.02
           + スタミナ補正
           # 回避系は別途確率判定で全有り無し
           # カウンター系も別途確率判定

final_dmg  = base_dmg
           * matrixA.dmg_mul    (ガード成功時 0、それ以外 1.0)
           * counter_mul        (カウンター成功時 1.3、失敗時 1.3 で被ダメに)
           * (atk.power / atk_zone == HEAD ? def.chin : def.gut)
```

## 4. ダメージ計算

§3末尾の式を参照。

- スタミナ0以下のとき: 威力 ×0.5、命中 -15%
- ガード成功時: 0ダメージ
- クリティカル: 命中時に確率発生で威力 ×1.6（フックは確率高め）

## 5. 距離感（5段階）

| 状態 | 説明 | 主な特徴 |
|---|---|---|
| クリンチ | 両者組み合った状態 | 短打のみ。レフェリー介入で「ブレイク」発生し離れる場合あり |
| 近距離 | 一歩で当てられる | アッパー有利 |
| 中距離 | 標準 | フック有利 |
| 遠距離 | 標準より遠い | ストレート有利 |
| 離れ気味 | アウトボクシング寄り | ジャブ・ストレート以外は届きにくい |

リング上の **ポジション**（中央/ロープ際/コーナー）は §2.7 で別管理。

### 距離の変化トリガー

- **ダッキング成功**: 1段近づく
- **スウェー成功**: 1段離れる
- **回避失敗**: 動かない
- ジャブヒット: 距離が広がる方向に1段
- フック空振り: 距離が広がる
- ストレート被弾: 後退
- 距離がクリンチで押し合いきると → 押された側の **ポジションが後退**（§2.7）

## 6. スタミナ・体力

| 指標 | 範囲 | 自然回復 |
|---|---|---|
| HEAD HP | 0〜100 | インターバル時のみ少量回復 |
| BODY HP | 0〜100 | 同上 |
| スタミナ | 0〜100 | 毎ターン +3 程度。インターバルで大回復 |
| KD カウント | 0〜3/R | 3でTKO。ラウンド開始時に0リセット |
| ポジション | 中央/ロープ際/コーナー | ラウンド開始時に中央リセット |

### chin と gut

```yaml
stats:
  power: float          # 攻撃倍率
  speed: int            # 行動順
  technique: int        # クリ・命中補正
  chin: int             # HEAD への被ダメ軽減
  gut: int              # BODY への被ダメ軽減
```

## 7. AI（敵）

- ステータスとパーソナリティでコマンド選択の重みづけが変わる
- スタミナ少 → ガード優先
- HP 少 → 一発逆転狙いでフック/アッパー確率上昇
- HEAD 残量が少ない相手には HEAD 攻撃比率を上げる
- 距離適性に応じて選択肢を変える
- ポジションが不利（ロープ際/コーナー）なら脱出寄り（スウェー優先）
