# 06 アセット方針

## 1. 現状（2026-05-10）

`resources/` 配下に **すでに揃っているアセット**：

```
resources/
├── background/
│   └── boxing_ring_battle_bg.png         (リング背景 1枚)
├── chara/
│   ├── 01_エリー_ヴィオラ.png            (立ち絵 standing 10名分)
│   ├── 01_エリー_ヴィオラ_data.json      (設定 10名分)
│   ├── 02_黒峰_サツキ.png
│   ├── 02_黒峰_サツキ_data.json
│   ├── ...
│   ├── 10_ハルカ_緑里.png
│   ├── 10_ハルカ_緑里_data.json
│   └── standing/                         (空フォルダ。今後ポーズ違いを置く想定)
└── samples/  ← 空。raw/に退避済み（権利物のため）
```

**まだ無いもの**:

- 各キャラの `_guard / _hit / _down` ポーズ（後で Codex で生成）
- アクション演出画像（`act_*.png`）
- UI 素材（HPバー枠、テキスト欄背景など）

## 2. 命名規則（2026-05-10 確定: アンダースコア統一）

```
キャラ立ち絵     : <id>_<姓>_<名>.png                   例) 01_エリー_ヴィオラ.png
キャラ設定       : <id>_<姓>_<名>_data.json
キャラ別ポーズ   : standing/<id>_<姓>_<名>_<pose>.png   例) standing/01_エリー_ヴィオラ_guard.png
背景             : bg_<scene>.png                       例) boxing_ring_battle_bg.png
アクション演出   : act_<type>.png                       例) act_hit_straight.png
UI               : ui_<part>.png                       例) ui_hp_frame.png
```

- 区切り文字は **必ずアンダースコア `_`**（中点 `・` ・スペース ` ` は使わない）
- 拡張子は **小文字**
- `<id>` は2桁ゼロ詰め

## 3. 画風（2026-05-10 確定: 美少女ボクシング寄り）

すでに10キャラが生成済みなので、これに合わせて **追加生成も同じトーン** で。

参考プロンプトのスタイル指示：

```
[Style] 2D Japanese anime/bishojo illustration, semi-realistic body proportions,
clean line art, cel-shaded with soft highlights, vivid saturated palette,
glossy skin tone, dramatic stage lighting from above, slight sweat glow.
[Subject] Female boxer, athletic toned body, sports bra/bikini-style top,
fitted shorts or trunks, boxing gloves, focused fighting expression.
[Composition] Isolated subject, transparent background, no border,
no text, no watermark, no logo.
```

> 既存10キャラと **同じキャラ** で別ポーズを生成する場合、`appearance` の髪・目・グローブ・トランクスの色をプロンプトに正確に転写する。

## 4. 必要素材リスト（v0 開始までに）

### 4.1. キャラ立ち絵（追加ポーズ）

10名 × 3ポーズ = 30枚（1人 standing は揃っているのでそれを除いた追加分）

| ポーズ | 用途 |
|---|---|
| `<id>_..._guard.png` | ガード姿勢、コマンド選択中の表示 |
| `<id>_..._hit.png` | 被弾、アクション再生中 |
| `<id>_..._down.png` | ダウン状態 |

> **TBD**: 全10キャラで全ポーズが必要か、まず2キャラ分（プレイヤー候補＋敵候補）だけ用意して動かすか

### 4.2. アクション演出（中央パネル用）

中サイズの **カットイン画像**。立ち絵を縮小して使うか、専用画像か検討中。

最小セット：

- `act_hit_jab.png` 衝撃線・スパーク
- `act_hit_straight.png` 強い放射状の衝撃
- `act_hit_hook.png` 横軌道の衝撃線
- `act_hit_upper.png` 下から上への衝撃
- `act_hit_crit.png` クリーンヒット用フラッシュ
- `act_block.png` 拳の衝突・盾型
- `act_miss.png` 風切り線
- `act_ko.png` "K.O.!" 大文字

### 4.3. UI 素材

- `ui_hp_frame_head.png` HEAD HPバー枠（グラデセグメント）
- `ui_hp_frame_body.png` BODY HPバー枠
- `ui_panel_command.png` 中央コマンドパネル背景
- `ui_panel_text.png` テキスト欄背景（半透明）
- `ui_button_*.png` 各種ボタン

> CSS / コードで描く案もあるので、画像化は最後でいい。

## 5. Codex に依頼するときのテンプレ

```
You are a 2D illustrator. Generate <FILE_NAME>.

[Style]
2D Japanese anime/bishojo illustration, semi-realistic body proportions,
clean line art, cel-shaded with soft highlights, vivid saturated palette,
glossy skin tone, dramatic stage lighting from above, slight sweat glow.

[Subject]
<キャラ名>, <髪の色とスタイル>, <目の色>, <グローブの色>,
<トランクスの色>, <ポーズ>(neutral/guard/hit/down), <表情>。
※ 必ず resources/chara/ にある standing と同一キャラに見えるよう、
  appearance フィールドを忠実に再現すること。

[Output]
- PNG with transparent background
- Resolution: 512x768 (character) / 512x512 (action) / 1280x720 (background)
- Subject centered, no cropping at the edges.

Negative prompt: text, watermark, signature, ugly, blurry, deformed,
extra fingers, extra limbs, low quality, different character.
```

## 6. 権利関係 / git 管理方針（2026-05-10 更新）

**`resources/` は丸ごと gitignore する** 方針に変更。

| フォルダ | git | 理由 |
|---|---|---|
| `resources/chara/` | **管理外** | キャラ画像はローカル運用、公開リポに載せない |
| `resources/background/` | **管理外** | 同上 |
| `resources/samples/` | **管理外** | 参考画像（自作UIモックや他作品スクショ） |
| `raw/` | 管理外 | 入力データ・参考画像置き場 |
| `work/` | 管理外 | 中間生成物 |
| `docs/` | コミット | 公開する設計書 |

### 同期方法

resources/ はリポに含めないので、別マシンで作業する場合は **手動コピー** か別途 OneDrive/Drive で同期する。

### サイズ感

- 1キャラ ~200KB × 10 = 2MB 程度
- 背景 ~1MB
- 立ち絵ポーズ追加が増えるとリポが太る恐れ → **gitignore済みで安心**
