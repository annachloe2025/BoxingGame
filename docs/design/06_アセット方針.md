# 06 アセット方針

## 1. 必要な画像（v0 の最小セット）

### キャラ立ち絵

各キャラに最低 4ポーズ：

- `<id>_neutral.png` ── 通常立ち
- `<id>_guard.png`   ── ガード姿勢
- `<id>_hit.png`     ── 被弾
- `<id>_down.png`    ── ダウン

例: `player_01_neutral.png` / `enemy_rocky_guard.png`

サイズ: 上半身〜腰くらい、512×768 px、背景透過 PNG

### カットイン用（中央パネル アクション再生時）

立ち絵をそのまま縮小して使う案と、専用の中サイズ画像を別に用意する案がある。

> **TBD**: 立ち絵を使い回すか専用画像を用意するか。前者なら工数小、後者なら見栄えが上がる。

### 背景

- `bg_arena.png` ── ボクシングリング＋観客＋照明
- 1280×720（→ 後で 1920×1080）

### UI 要素

- HEAD/BODY HP バー枠
- 中央パネル枠
- テキスト欄背景
- ボタン素材

> **TBD**: UI を画像で作るか CSS / シェーダで作るか（エンジンによる）

## 2. 画像生成方針

ユーザー方針: **画像は Codex に作ってもらう**。

具体的には：

1. `docs/design/06_アセット方針.md` または別ファイルにプロンプトを集約
2. Codex（または ChatGPT 系）に「このプロンプトでこのファイル名で生成して」と指示
3. 生成された画像を `art/<カテゴリ>/<ファイル名>.png` に配置
4. ゲームから参照

### プロンプトのスタイル統一

```
[Style] 2D anime/illustration style, semi-realistic, dramatic lighting,
clean line art, cel-shaded, vivid but slightly desaturated palette,
isolated subject, transparent background, no border, no text, no watermark.
```

> **TBD**: 画風（美少女路線 / 少年漫画路線 / その他）が決まったら、
> ここのスタイル指示を最終化する。

## 3. ファイル命名規則

```
キャラ立ち絵     : <role>_<name>_<pose>.png
                   例) player_01_neutral.png / enemy_rocky_hit.png
カットイン       : <role>_<name>_cutin_<pose>.png   (専用画像を作る場合)
背景             : bg_<scene>.png                例) bg_arena.png
アクション演出   : act_<type>.png                  例) act_hit_straight.png
UI               : ui_<part>.png                  例) ui_hp_frame.png
```

すべて英小文字＋アンダースコア。日本語ファイル名は使わない。

## 4. フォルダ配置（実装時）

```
art/
├── characters/
│   ├── player_01_neutral.png
│   ├── player_01_guard.png
│   ├── ...
│   └── enemy_rocky_*.png
├── action/
│   ├── act_hit_jab.png
│   ├── act_hit_straight.png
│   └── ...
├── bg/
│   └── bg_arena.png
└── ui/
    ├── ui_hp_frame.png
    └── ui_panel.png
```

エンジン（Unity / Godot）に取り込むときは、それぞれの規約に沿って配置するが、ファイル名は同じものを保つ。

## 5. 権利関係

- 参考画像（他社製品のスクリーンショット等）は **絶対にコミットしない**
- `raw/` は `.gitignore` 済み。参考画像はそこに置いて議論にだけ使う
