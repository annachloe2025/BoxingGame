## 1.5. ステータスの寄与（どのステがどの計算に効くか）

### PHYSICAL

| ステータス | 効く計算 | 効き方 | 備考 |
|---|---|---|---|
| **head_hp** | HEAD ダメージ蓄積上限 | 0 になるとダウン発生（KD+1） | 立ち上がり率にも影響（残量比率） |
| **body_hp** | BODY ダメージ蓄積上限 | 0 になるとダウン発生（KD+1） | 同上 |
| **punch_power** | ダメージ計算（powMod） | `dmg ×= power / 14` | キャラ全体のパンチ威力倍率 |
| **endurance** | ダメージ計算（endMod） | `dmg ×= 14 / endurance` | 全部位の被ダメ軽減（chin/gut の元値） |
| **stamina** | スタミナ最大値 / 回避率補正 | 最大値そのまま、`evadeChance += (def.sta - atk.sta) × 0.005`、スタ不足時は `dmg ×= 0.5` & `hit -= 15` | 連戦・連撃に強くなる |
| **toughness** | 立ち上がり率 | `getUp ×= toughness / 14` | ダウンしても起きやすくなる |
| **counter** | カウンター成功率 | `chance += (counter - 14) × 0.02` | カウンター系防御の上手さ |

### PUNCH（技ごと熟練度）

| ステータス | 効く計算 | 効き方 |
|---|---|---|
| **\<tech\>_atk** | 命中率／ダメージ | `hit += (atk_atk - def_def) × 1.5`、`dmg ×= atk_atk / 14` |
| **\<tech\>_def** | 命中率／回避率／カウンター成功率 | `hit -= (atk_atk - def_def) × 1.5` (= 命中を下げる)、`evadeChance += def_def × 0.005`、`counterChance += def_def × 0.005` |

> 例: `jab_atk` が高いキャラは「ジャブの命中とダメージ両方上がる」、
> `hook_def` が高いキャラは「フックを食らいにくく、フックに対するカウンター/回避もしやすい」

### 効果が**まだ無い**ステータス（要件確認）

| ステータス | 現状 | 検討事項 |
|---|---|---|
| `<tech>_def` | ガード成功時のダメージ減には効いていない（ガードはアタリ判定で 0 か通常dmg のみ） | 「ガード成功時の被ダメも `def_def / 14` で割る」を入れるか |
| `endurance` | 全部位共通の倍率。HEAD/BODY 別の chin/gut にはまだ分岐していない | 設計書 03 では chin/gut 分離だったが実装では未分離 |
| `weight_class`（階級） | 戦闘ロジックには未影響 | 階級差で何かペナルティを入れるか |
| スキル（パッシブ）/ 必殺フロー | 未実装 | Phase 3 で追加予定 |

## 2. 技の基準値（TECHS）

| 技 | base_dmg | base_hit | speed | sta（消費） |
|---|---|---|---|---|
| jab | 5 | 80% | 8 | 5 |
| straight | 11 | 65% | 5 | 12 |
| hook | 18 | 50% | 3 | 18 |
| upper | 22 | 60% | 3 | 20 |

- `base_dmg` … 基準ダメージ（パンチ力等で倍率がかかる）
- `base_hit` … 基準命中率（距離・熟練度で補正）
- `speed` … 先攻判定に影響
- `sta` … 1回消費するスタミナ

## 3. 距離適性マトリクス（DIST_MOD）

距離は5段階: `[0:クリンチ, 1:近, 2:中, 3:遠, 4:離れ]`
※ ロープ際/コーナー際は別概念（リングポジション）

| 技 | クリンチ | 近 | 中 | 遠 | 離れ |
|---|---|---|---|---|---|
| jab | -10 | 0 | 0 | 0 | 0 |
| straight | -30 | -10 | 0 | **+15** | -10 |
| hook | -10 | 0 | **+15** | -10 | -30 |
| upper | 0 | **+15** | 0 | -20 | -40 |

### EVADE_RATE（ダッキング/スウェー）

| 攻撃部位 | 回避基準率 |
|---|---|
| HEAD | 0.55 |
| BODY | 0.25 |

### COUNTER_RATE（カウンター）

| 攻撃部位 | 成功基準率 |
|---|---|
| HEAD | 0.30 |
| BODY | 0.50 |

## 5. 主要な計算式

### 5.1 命中率（hit %）

```
hit = base_hit                                     # 80/65/50/60
    + DIST_MOD[tech][distance]                     # -40 〜 +15
    + (atk[tech]_atk - def[tech]_def) * 1.5        # 技熟練度差
    + POS_PENALTY_HIT[atk.pos]                     # 攻撃側ポジション
    - 15  if atk.sta < tech.sta                    # スタミナ不足

hit = clamp(hit, 5, 95)
isHit = random()*100 < hit
```

### 5.2 ダメージ計算

```
powMod  = atk.punch_power / 14
techMod = atk[tech]_atk    / 14
endMod  = 14 / def.endurance

dmg = base_dmg * powMod * techMod * endMod
if atk.sta < tech.sta:  dmg *= 0.5
dmg = max(1, round(dmg))
```

> 例: ナギサのフックHEAD（base_dmg=18）で平均的な相手に当てると
> `18 × (18/14) × (17/14) × (14/14) ≈ 28 ダメ`

### 5.3 ダッキング/スウェーの回避率

```
chance = EVADE_RATE[off.zone]                      # HEAD:0.55 / BODY:0.25
       + def[tech]_def * 0.005                     # 防御熟練度
       + (def.stamina - atk.stamina) * 0.005       # スタミナ差
       + POS_PENALTY_EVADE[def.pos]

chance = clamp(chance, 0.05, 0.90)
```

成功 → 攻撃を完全に避け、距離が1段移動（ダッキング=-1, スウェー=+1）
失敗 → 通常の命中判定が走る、距離変化なし

### 5.4 カウンター成功率

```
chance = COUNTER_RATE[off.zone]                    # HEAD:0.30 / BODY:0.50
       + (def.counter - 14) * 0.02                 # カウンターステ補正
       + def[tech]_def * 0.005                     # 防御熟練度
       + POS_PENALTY_EVADE[def.pos]

chance = clamp(chance, 0.05, 0.95)
```

| 結果 | ダメージ |
|---|---|
| 成功 | 攻撃側に `dmg × 1.3` |
| 失敗 | 防御側に `dmg × 1.3` |

### 5.5 ガード判定

```
攻撃部位 == ガード部位（HEAD vs HEAD or BODY vs BODY） → 被ダメ 0
攻撃部位 != ガード部位（読み外し）                    → 通常の命中判定 + 通常dmg
```

クリーンヒットボーナスは **未搭載**。

### 5.6 先攻判定

```
speedDiff = (atk.stamina + tech.speed) - (def.stamina + tech.speed)
pFirstChance = clamp(0.5 + speedDiff * 0.025, 0.1, 0.9)
playerFirst = random() < pFirstChance
```

### 5.7 立ち上がり率（ダウン時）

```
base   = 0.7
buffer = head_hp_max * 0.10

getUp = base
      * max(0.05, (head_hp + buffer) / head_hp_max)   # HEAD残量比
      * max(0.05, (body_hp + buffer) / body_hp_max)   # BODY残量比
      * (toughness / 14)                              # タフネス補正
      * (1 - (round - 1) * 0.1)                       # 後ラウンド-10%

getUp = clamp(getUp, 0.05, 0.95)
```

成功時: `head_hp += head_hp_max * 0.30`、`body_hp` も同様に回復
3KD（1ラウンド3回ダウン） → TKO負け
カウントアウト（10カウント） → KO負け

## 6. ターン・ラウンド構造

### 1ターンの流れ

```
プレイヤー: オフェンスX + ディフェンスY
敵:        オフェンスZ + ディフェンスW
        ↓ 同時公開（速度＋確率で先攻判定）
解決A: プレイヤーのX vs 敵のW
解決B: 敵のZ vs プレイヤーのY
        ↓
HP / 距離 / リングポジション / KD反映
スタミナ +3
```

### ラウンド構造

| 項目 | 値 |
|---|---|
| 1ラウンド | 12ターン（= 3:00、1ターン=15秒換算） |
| ラウンド数 | 3 |
| インターバル時の回復 | HEAD/BODY **+20**、スタミナ全回復、KD 0リセット、距離=中、ポジション=中央 |
| 最終判定 | 3R終了時、`(head_hp + body_hp)` 合計が大きい方の勝ち |

## 7. 設計書 03 との差分

設計書 03 の方針と、実装の数値が異なる箇所：

| 項目 | 設計書03 | 実装（08） |
|---|---|---|
| スピード補正 | `* 0.02` | `* 0.025` |
| カウンターBODYボーナス | "+15〜20%" | base_rate を直接 0.30/0.50 |
| クリティカル | 命中時に確率発生で威力 ×1.6（フックは確率高） | **未実装** |
| 立ち上がり率の式 | 概念のみ記述 | §5.7 で具体化 |
| 距離適性ペナルティ | 概要のみ | §3 で具体マトリクス |

→ 実装が「正本」になっているので、03 を最終的にこの数値に合わせて更新するか、
プロトの数値を 03 に近づけるかは要決定。
